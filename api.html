POST /api/v2.1/deployment-updates/(string: id)/update/(string: phase)¶

Provides support for two phases of deployment update. The phase is chosen according to the phase arg, and the id is used by this step.

In the first phase the deployment update is 1. Staged (from a new blueprint) 2. The steps are extracted and saved onto the data model. 3. The data storage is manipulated according to the addition/modification steps. 4. The update workflow is run, executing any lifecycles of add/removed nodes or relationships.

The second step finalizes the commit by manipulating the data model according to any removal steps.

In order :param id: for the initiate step it’s the deployment_id, and for the finalize step it’s the update_id :param phase: initiate or finalize :return: update response
